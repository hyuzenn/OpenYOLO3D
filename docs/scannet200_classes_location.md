## Inference-time class set used by OpenYOLO3D

`pretrained/config_scannet200.yaml` lists `network2d.text_prompts` (199
entries — `CLASS_LABELS_200` minus `wall`/`floor`, with one duplicate). The
runtime appends a single `' '` background prompt:

```
utils/utils_2d.py:38 → self.texts = [[t] for t in config["network2d"]["text_prompts"]] + [[' ']]
utils/__init__.py:104 → self.num_classes = len(text)+1
```

So OpenYOLO3D classifies into (199 instance classes) + (1 background) at
inference time, even though the eval set is the full 200.

## CLIP text encoder

OpenYOLO3D doesn't run a standalone CLIP encoder — it uses YOLO-World v2-X,
whose text branch is HuggingFace CLIP:

| What                              | Where                                                                   |
|-----------------------------------|-------------------------------------------------------------------------|
| `HuggingCLIPLanguageBackbone`     | `models/YOLO-World/yolo_world/models/backbones/mm_backbone.py:60`       |
| Tokenizer                         | `transformers.AutoTokenizer.from_pretrained(model_name)`                |
| Encoder                           | `transformers.CLIPTextModelWithProjection.from_pretrained(model_name)`  |
| L2 normalization                  | `mm_backbone.py:93` — `txt_feats / txt_feats.norm(p=2, dim=-1, keepdim=True)` |
| Model name (config)               | `pretrained/configs/yolo_world_v2_x_vlpan_bn_2e-3_100e_4x8gpus_obj365v1_goldg_train_lvis_minival.py:33` |

Resolved `model_name`: **`openai/clip-vit-base-patch32`** — embed dim 512.

The text-tower output used is `text_embeds` from
`CLIPTextModelWithProjection` (i.e., the projection-head embedding, not the
last hidden state's pooler output).

Cache present locally at:
`~/.cache/huggingface/hub/models--openai--clip-vit-base-patch32` — no
download needed at extraction time.

## Prompt template

OpenYOLO3D feeds **bare class names** (no "a photo of a {…}" wrapper). Source
of truth: `pretrained/config_scannet200.yaml`'s `network2d.text_prompts`,
which lists e.g. `'chair'`, `'office chair'`, `'kitchen cabinet'` directly.

`method_scannet/extract_prompt_embeddings.py` mirrors this with
`prompt_template = "{class_name}"` and saves `prompt_template` into the
`.pt` payload so consumers can re-derive it.
